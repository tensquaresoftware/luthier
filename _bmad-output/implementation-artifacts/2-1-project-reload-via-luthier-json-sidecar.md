# Story 2.1: Project Reload via `.luthier.json` Sidecar

Status: done

<!-- Note: Validation is optional. Run validate-create-story for quality check before dev-story. -->

## Story

As a JUCE developer,
I want Luthier to reload my project configuration from the `.luthier.json` sidecar,
so that all fields are restored exactly as I configured them, without fragile CMake text parsing.

## Acceptance Criteria

1. **Given** a project generated by Luthier (containing `.luthier.json`), **when** I click "Open Project…" and select the folder, **then** `project_reader.read_project()` reads `.luthier.json` and returns a `ProjectSpec` with all fields intact.
2. **Given** the loaded `ProjectSpec` is applied to the UI, **when** I click "Generate Project" without changing any field, **then** the generated files produce an empty diff vs. the originals (byte-for-byte round-trip fidelity).
3. **Given** `.luthier.json` exists but contains malformed JSON, **when** `read_project()` is called, **then** it returns `None` — it does not attempt regex fallback and does not silently load partial data.
4. **Given** `read_project()` returns `None` for any reason, **when** `MainWindow._on_open()` handles the result, **then** an error dialog is displayed; the form retains its previous state unchanged.

### Current State of `core/project_reader.py`

```35:44:core/project_reader.py
def read_project(project_dir: Path) -> Optional[ProjectSpec]:
    cmake = project_dir / "CMakeLists.txt"
    if not cmake.exists():
        return None
    values = _parse_cmakelists(cmake.read_text(encoding="utf-8"))
    if values is None:
        return None
    values["destinationDir"] = str(project_dir.parent)
    values.update(_parse_build_settings(project_dir))
    return ProjectSpec.from_dict(values)
```

**Must become:**

```python
def read_project(project_dir: Path) -> Optional[ProjectSpec]:
    sidecar = project_dir / _SIDECAR
    if sidecar.exists():
        return _read_sidecar(sidecar, project_dir)
    return _read_from_cmake(project_dir)
```

**Critical invariant (AC3):** When `.luthier.json` **exists**, never fall through to `_read_from_cmake` — even if CMake is valid. A corrupt sidecar must return `None`.

### Sidecar JSON Format

Written by `ProjectWriter._write_sidecar()`:

```69:73:core/project_writer.py
    def _write_sidecar(self, dest: Path, spec: ProjectSpec) -> None:
        (dest / ".luthier.json").write_text(
            json.dumps(spec.to_dict(), indent=2, ensure_ascii=False),
            encoding="utf-8",
        )
```

Keys are **camelCase** (matching `ProjectSpec.to_dict()`): `projectName`, `projectDisplayName`, `pluginType`, `pluginFormats`, `copyToSystemFolders`, etc.

**Not in sidecar (by design — AD-7):** `juceDir` — lives in `Preferences` only, injected at generation via `ProjectGenerator.generate(spec, juce_dir=...)`.

**Not in sidecar (by design — AD-9):** template override paths — resolved at write time from `templates_store.overrides_dir()`.

### `destinationDir` Override on Read

Both read paths must set `destinationDir = str(project_dir.parent)` after deserialising. The opened folder's parent is authoritative — if the user moved the project directory, destination should reflect the new location. This matches the existing CMake reader and enables correct regeneration at the new path.

### Round-Trip Fidelity (AC2) — Known Diff Sources

For load → regenerate without edits to produce an **empty diff**, these conditions must hold:

| Factor | In sidecar? | Round-trip impact |
|--------|-------------|-------------------|
| All `ProjectSpec` fields | ✅ Yes | Restored exactly via sidecar |
| `juceDir` | ❌ Preferences only | If `Preferences.juce_dir` changed between generate and re-generate, `CMakeLists.txt` diff on `set(JUCE_DIR ...)` line — **expected**, not a story 2-1 bug |
| Template overrides (.gitignore, Source/*) | ❌ Resolved at write | If user changed overrides in Templates tab between load and regen, source files / `.gitignore` may differ — **expected** |
| `projectDisplayName` empty-string normalisation | ✅ Handled at generate | `ProjectPage.spec()` normalises empty display name → project name **before** generation, so sidecar stores the normalised value. No special handling needed on read. |

**Test strategy for AC2:** Use programmatic generate → read → regenerate in `tests/test_story_2_1.py` with **no template overrides** and **empty `juce_dir`**. Compare all files byte-for-byte including `.luthier.json`.
