- (event) Architecture run started — coaching mode, brownfield, feature altitude. Deliverables: spine + markdown explanation doc. Goal: evaluate current state, strengthen for feature goals (round-trip, tests, JUCE path, CLAP).
- (constraint) Stack fixed: Python 3.14, PySide6 ≥ 6.7, no external runtime deps beyond PySide6, stdlib only. [ADOPTED]
- (constraint) Clean-code hard limits in force: 15-line functions, 200-line classes, ≤3 params, cyclomatic complexity < 5, ≤2 indentation levels. [ADOPTED]
- (decision) AD-1 — Central data model: introduce ProjectSpec dataclass in core/project_spec.py as the unique contract between all layers (form, reader, render_context, preferences). Binds: all. Prevents: key-mismatch bugs between layers, silent failures from untyped dicts. Rule: no layer passes a raw dict of project data — all cross-layer exchange uses ProjectSpec. Fields use snake_case (Python convention); CMake template keys stay camelCase (CMake convention). [ADOPTED after coaching]
- (decision) AD-2 — ProjectSpec is the single data model: artefact config (copy_to_system_folders, copy_to_artefacts_dir, artefacts_dir_windows/macos/linux) merges into ProjectSpec alongside identity fields. Binds: ProjectPage, ProjectGenerator, ProjectWriter, project_reader, Preferences. Prevents: two-dict split that forces callers to assemble ProjectSpec from two incompatible halves. Rule: generator.generate() takes a single ProjectSpec; ProjectPage exposes a single spec() method replacing values()+config(). [ADOPTED after coaching]
- (decision) AD-3 — Round-trip strategy: sidecar .luthier.json written at generation time, read first by project_reader. Fallback to regex CMake parsing for legacy/hand-edited projects. Binds: ProjectWriter, project_reader. Prevents: regex fragility breaking round-trip when CMake is reformatted or hand-edited. Rule: ProjectWriter always writes .luthier.json containing the full ProjectSpec; project_reader reads .luthier.json first, falls back to _parse_cmakelists() if absent. [ADOPTED after coaching]
- (decision) AD-4 — Atomic project write: ProjectWriter writes to a sibling temp dir (project_dir.parent / project_name + '.tmp'), then renames to final path. On error, temp dir is cleaned up and original is untouched. Binds: ProjectWriter. Prevents: half-written corrupted project on generation error. Rule: _reset_project_dir() is replaced by a write-to-temp + rename pattern; the final rename is the commit point.
- (decision) AD-5 — Preferences save-on-update: Preferences.save() is called immediately after every update() that changes persisted data. Binds: MainWindow, Preferences. Prevents: preference drift where in-memory state diverges from disk after open/generate. Rule: two call sites — after successful generate, after successful open — each call prefs.update(spec) then prefs.save().
- (decision) AD-6 — Test strategy: pytest in tests/unit/ (pure functions, no I/O) and tests/integration/ (round-trip with tmp_path fixture). pytest added to requirements-dev.txt as dev dependency. No Qt tests. Priority: core/ validators, render_context, the full ProjectSpec round-trip. Binds: tests/. Prevents: untested pure functions and fragile regex going undetected. Rule: every public function in core/ that takes no Qt dependency has a corresponding unit test.
- (decision) AD-7 — juce_dir is Preferences-only: not a field on ProjectSpec. juce_dir describes the dev environment, not the plugin project. It is read from Preferences and injected into CMakeLists.txt at generation time without being part of the round-trip data model. Binds: ProjectSpec, Preferences, render_context. Prevents: per-project JUCE path drift and unnecessary sidecar bloat.
- (decision) Deferred — CLAP format support: add CLAP to FormatsPage checkbox list and CMakeLists.txt template. No new architectural decision required; plugin_formats field on ProjectSpec already carries it as a space-separated string. Implement when ready.
- (event) Coaching phase complete. All major architectural decisions validated. Moving to Finalize.
- (event) Spine and explanation doc written. Proceeding to reviewer gate.
- (decision) AD-9 — User template overrides resolved at write time, not stored in ProjectSpec. ProjectWriter resolves via templates_store.overrides_dir() at construction. ProjectSpec carries no template reference. Prevents two incompatible representations diverging.
- (event) Reviewer gate complete. 3 gaps closed: AD-3 clarified sole deserialiser + partial fallback = None; AD-4 clarified overwrite semantics; AD-5 clarified save is app-layer only; AD-9 added for template override ownership. Spine set to final.
